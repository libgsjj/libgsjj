3 # Deterministic finite automata learning library
4 The `libgsjj` is an open-source framework for learning deterministic finite automata. This project was done by Jonathan Joertz and GaÃ«tan Staquet (we named this library after our initials in our humility) for our Master's project at the University of Mons (UMONS).

6 This README uses Doxygen's features. Please see the [online documentation](https://libgsjj.github.io/libgsjj/index.html) for a easier to read README.

8 ## Learning deterministic finite automata
9 This library focuses on passive learning algorithms. In other words, the learning task is the following:
10  1. We have a sample \f$S = (S_+, S_-)\f$ over an alphabet \f$\sigma\f$ where \f$S_+\f$ is the finite set of words the DFA must accept and \f$S_-\f$ is the finite set of words the DFA must reject.
11  2. We must build the smallest DFA consistent with \f$S\f$. A DFA \f$A\f$ is consistent with \f$S\f$ if \f$S_+ \subseteq L(A) \land S_- \cap L(A) = \emptyset\f$.
12 
13 To build such a DFA, we use what we call passive learning algorithms. The following algorithms are implemented in `libgsjj`:
14  - Biermann and Feldman's algorithm
15  - Grinchtein, Leucker and Piterman's unary and binary algorithms
16  - Heule and Verwer's algorithm
17  - Neider and Jansen's algorithm

19 ## How to use libgsjj

35 ### How to compile
36 When you are in the root directory of this library:
37 ```bash
38 mkdir build && cd build && cmake .. && make
39 ```
40 
41 This will build the library. If you want to install it, use `make install`. You can change the installation folder with `cmake -DCMAKE_INSTALL_PREFIX=your/path ..` (or use a graphical interface for CMake).
42 
43 To enable building the benchmarks, instead of `cmake ..` use `cmake -DBUILD_BENCHMARKS=TRUE ..`.
44 
45 To enable building the tests, instead of `cmake ..` use `cmake -DBUILD_TESTS=TRUE ..`. You can of course enable the benchmarks and the tests at the same time with `cmake -DBUILD_BENCHMARKS=TRUE -DBUILD_TESTS=TRUE ..`
46 
47 ### How to use in a project
48 If you use CMake, it's easy. The building stage described before also creates a CMake Config file. Therefore, in your `CMakeLists.txt`, you only have to add the line
49 ```cmake
50 find_library(gsjj)
51 ```
52 and `gsjj` will be available a target you can link against. It's possible the user must set the `gsjj_DIR` CMake parameter to be able to build your program if the files are not installed in the standard location.
53 
54 To construct a method, please see the gsjj::passive::constructMethod functions. Here is the complete list of built-in methods with the names used in the program:
55  * Biermann and Feldman: `biermann`
56  * Neider and Jansen: `neider`
57  * Grinchtein, Leucker and Piterman unary:
58  * CNF: `unary`
59  * Non-CNF: `unaryNonCNF`
60  * Grinchtein, Leucker and Piterman binary:
61  * CNF: `binary`
62  * Non-CNF: `binaryNonCNF`
63  * Heule and Verwer:
64  * CNF: `heule`
65  * Non-CNF: `heuleNonCNF`

67 #### How to add a new method
68 Let's say we want to create a new method called `TestMethod`. It's really simple. We just have to create a new class, makes it inheriting gsjj::passive::Method and gsjj::passive::RegisterInFactory<TestMethod>. Here is a very simple (and stupid) example:
69 ```cpp
70 class TestMethod :
71  public gsjj::passive::Method,
72  public gsjj::passive::RegisterInFactory<TestMethod> {
73  public:
74  bool solve() override {
75  // Every method inheriting RegisterInFactory must have a line using s_registered
76  s_registered = s_registered;
77  return true;
78  }
79  bool hasSolution() const override {
80  return true;
81  }
82  std::unique_ptr<gsjj::DFA<char>> constructDFA() override {
83  return nullptr;
84  }
85  void printVariables() const override {
86  }
87  void setStopTrigger(const std::chrono::seconds &timeLimit, std::atomic_bool &stopTrigger, const bool *stopPointer) override {
88  // If the method relies on CVC4, use timeLimit to set the engine's timeLimit
89  // If the method relies on Maple, use stopTrigger
90  // If the method relies on Limboole, use stopPointer
91  }
92  static std::string getFactoryName() {
93  return "test";
94  }
95 
96  protected:
97  TestMethod(
98  const std::set<std::string> &SpSet,
99  const std::set<std::string> &SmSet,
100  const std::set<std::string> &SSet,
101  const std::set<std::string> &prefixesSet,
102  const std::set<char> &alphabetSet,
103  unsigned int n) :
104  Method(SpSet, SmSet, SSet, prefixesSet, alphabetSet, n) {}
105 };
106 ```
